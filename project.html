Weather App (Python)


I initially conceived of this project as a fun way to explore some of the many packages available in the Python ecosystem. While I had experimented with some of them individually in the past, this project was a great opportunity to bring together different elements- building a GUI, scraping data from an external website and parsing it into a readable format that would integrate with my application.

I used tkinter to build the GUI. The first tab ("Weather Conditions") allows the user to enter a four-digit station code used by weather.gov to get the latest data for that station. Of course, my default station is set to Central Park, New York, NY - but you can get data for any location in the US! In order to scrape the data, I used urllib.request, which offered helpful functions to parse the data into a useful format.

However, I wasn't satisfied with this from a user perspective. After all, it seemed unrealistic to ask the user to memorize a station code- or worse, to have to look for it online (eliminating the need for this app in the first place).

The natural solution was, of course, to provide the user with a list of station codes to choose from. Since the station codes are only four-digits, I needed to provide a meaningful description of the station- e.g. New York Kennedy International Airport for KJFK.

Since there were thousands of codes in total, I decided to break down each code by State- this would also allow the user to search more easily for the station they wanted. This was where html.parser proved useful- I was able to look for the regex representing the start of each station code and its corresponding city description, and add these to a list for the main function to use.
